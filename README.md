# E-Commerce Backend

A REST API backend for an e-commerce platform, built with **Node.js**, **Express 5**, **PostgreSQL**, and **Drizzle ORM**. It covers the core commerce flow — authentication, product catalog, cart, addresses, orders, wishlists, reviews, and image uploads — with role-based access for regular users and admins.

## Features

- **Authentication** — signup / login with hashed passwords (bcrypt) and JWT-based auth.
- **Products & Categories** — full CRUD; public read access, admin-only writes.
- **Cart** — add, update, view, and remove cart items per user.
- **Addresses** — manage multiple shipping addresses per user, with a default flag.
- **Orders** — place orders from the cart; users view their own orders, admins manage all orders and update status.
- **Wishlist** — save and remove favorite products.
- **Reviews** — product ratings and comments.
- **Image Uploads** — file uploads to Cloudinary via Multer.
- **Role-based access control** — `user` and `admin` roles enforced by middleware.

## Data Model

The schema (see [src/db/schema/schema.js](src/db/schema/schema.js)) defines these tables:

- **users** — id, name, email (unique), password, role (`user` / `admin`)
- **categories** — name, slug, description, image
- **products** — name, slug, sku, price, discount_price, stock, thumbnail, `category_id` → categories
- **product_images** — additional images per product
- **addresses** — shipping details per user, `is_default` flag
- **carts** / **cart_items** — one cart per user, items reference products
- **orders** / **order_items** — orders with `payment_status` and `order_status`, line items with captured price
- **wishlist** — user ↔ product favorites
- **reviews** — rating + comment per user/product

All primary keys are UUIDs generated by the database.

## Getting Started

### Prerequisites

- Node.js 18+
- Docker (for the local PostgreSQL container) or an existing PostgreSQL instance
- A Cloudinary account (for image uploads)

### 1. Install dependencies

```bash
npm install
```

### 2. Configure environment variables

Create a `.env` file in the project root:

```env
DATABASE_URL=postgres://postgres:mypass@localhost:5432/e-commerce
PORT=8000
JWT_SECRET_KEY=your_jwt_secret

CLOUDINARY_CLOUD_NAME=your_cloud_name
CLOUDINARY_API_KEY=your_api_key
CLOUDINARY_API_SECRET=your_api_secret
```

> The `DATABASE_URL` above matches the credentials in `docker-compose.yml`. Adjust it if you use your own database.

### 3. Start PostgreSQL

Using Docker:

```bash
docker compose up -d
```

This starts a `postgres` container on port `5432` with database `e-commerce` and a persistent volume.

### 4. Run migrations

Generate and apply the schema to your database with Drizzle Kit:

```bash
npx drizzle-kit generate   # create SQL migrations from the schema
npx drizzle-kit migrate    # apply migrations to the database
```

### 5. Start the server

```bash
npm run dev    # watch mode (auto-restart on changes)
# or
npm start      # plain start
```

The server runs on `http://localhost:8000` (or your `PORT`). Verify it's up:

```bash
curl http://localhost:8000/health
# "server is running fine"
```

## API Overview

Base URL: `http://localhost:8000`

Protected routes expect an `Authorization: Bearer <token>` header. Routes marked **Admin** additionally require the `admin` role.

### Auth — `/auth`
| Method | Path | Description |
|--------|------|-------------|
| POST | `/auth/signup` | Register a new user |
| POST | `/auth/login` | Log in, returns a JWT |

### Categories — `/categories`
| Method | Path | Access |
|--------|------|--------|
| GET | `/categories/categories` | Public |
| GET | `/categories/categories/:id` | Public |
| POST | `/categories/categories` | Admin |
| PATCH | `/categories/categories/:id` | Admin |
| DELETE | `/categories/categories/:id` | Admin |

### Products — `/product`
| Method | Path | Access |
|--------|------|--------|
| GET | `/product` | Public |
| GET | `/product/:id` | Public |
| POST | `/product` | Admin |
| PATCH | `/product/:id` | Admin |
| DELETE | `/product/:id` | Admin |

### Addresses — `/address` (auth required)
| Method | Path |
|--------|------|
| GET | `/address` |
| GET | `/address/:id` |
| POST | `/address` |
| PATCH | `/address/:id` |
| DELETE | `/address/:id` |

### Cart — `/cart` (auth required)
| Method | Path | Description |
|--------|------|-------------|
| GET | `/cart` | Get current user's cart |
| POST | `/cart` | Add item to cart |
| PATCH | `/cart` | Update item quantity |
| DELETE | `/cart` | Remove item |

### Orders — `/orders` (auth required)
| Method | Path | Access |
|--------|------|--------|
| GET | `/orders` | User (own orders) |
| GET | `/orders/:orderId` | User |
| POST | `/orders` | User (place order) |
| GET | `/orders/admin/all` | Admin |
| PATCH | `/orders/admin/:orderId` | Admin (update status) |

### Wishlist — `/wishlist` (auth required)
| Method | Path |
|--------|------|
| GET | `/wishlist` |
| POST | `/wishlist` |
| DELETE | `/wishlist/:product_id` |

### Upload — `/upload`
| Method | Path | Description |
|--------|------|-------------|
| POST | `/upload` | Upload an image (form field `image`), stored on Cloudinary |

### Health
| Method | Path |
|--------|------|
| GET | `/health` |
